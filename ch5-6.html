Von Neumann computer architecture, 4 major subsystems:

  1. Central Processing Unit (CPU): The CPU is the heart of the computer and performs all arithmetic and logic operations. It also fetches instructions from memory, decodes them, and executes them.
  2. Memory Unit: The memory unit stores both data and instructions that are currently being executed by the CPU. It is divided into two types of memory: primary memory (also called main memory) and secondary memory (such as hard drives and solid-state drives).
  3. Input/Output (I/O) Unit: The I/O unit handles all communication between the computer and its external devices, such as keyboards, mice, displays, printers, and network interfaces.
  4. System Bus: The system bus is the communication pathway between the CPU, memory, and I/O units. It is used to transfer data, instructions, and control signals between these components.

Von Neumann computer architecture, 3 characteristics:

  1. Stored Program Concept: In the Von Neumann architecture, both instructions and data are stored in the same memory space. This means that instructions can be treated as data and manipulated by the CPU. This concept allows the computer to be programmed to perform different tasks and enables the computer to be versatile.
  2. Sequential Execution: The Von Neumann architecture executes instructions sequentially, meaning that each instruction is executed one after the other in the order in which they are stored in memory. The CPU fetches an instruction from memory, decodes it, and then executes it before moving on to the next instruction. This sequential execution of instructions is also known as the fetch-decode-execute cycle.
  3. Single Control Unit: The Von Neumann architecture has a single control unit that is responsible for coordinating and controlling the flow of data and instructions between the different subsystems of the computer. The control unit is responsible for fetching instructions from memory, decoding them, and executing them in the correct sequence. This simplifies the design of the computer and makes it easier to understand and program.

Computer Architecture Terms

  1. RAM: A computer memory that is used to temporarily store data and instructions that are being actively used by the CPU.
  2. Cache: A type of high-speed memory that is used to temporarily store frequently accessed data or instructions.
  3. Principal of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. Temporal Locality: This sub-principle states that data or instructions that have been accessed recently are likely to be accessed again in the near future. For example, a loop in a program will often access the same data or instructions repeatedly, so the CPU can store this data or instruction in a cache to speed up subsequent accesses.
    2. Spatial Locality: This sub-principle states that data or instructions that are located near each other in memory are likely to be accessed together. For example, an array in a program will often access adjacent elements in the array, so the CPU can prefetch these elements into a cache to speed up subsequent accesses.
  4. ROM: A type of computer memory that is used to store data or instructions that cannot be changed or modified once they have been programmed into the memory.
  5. Memory cell size: The memory cell size refers to the amount of data that can be stored in a single memory cell in a computer's memory.
  6. Memory cell address: A unique identifier assigned to each memory cell in a computer's memory.
  7. Maximum memory size: The largest amount of memory that a computer system can support.
  8. Register: A small amount of high-speed memory that is located within the CPU.
  9. Fetch: The process by which the CPU retrieves an instruction from memory and loads it into a register for execution.
  10. Store: The process by which the CPU saves data or instructions back to memory.
  11. Memory access time: The amount of time it takes for the CPU to retrieve data or instructions from memory.
  12. DASD: DASD stands for Direct Access Storage Device, which is a type of computer storage device that allows for random access to data stored on a magnetic or optical disk.
  13. Arithmetic and Logic Unit (ALU): A component of the CPU that is responsible for performing arithmetic and logical operations.
  14. Control Unit: A component of the CPU that is responsible for controlling the flow of data and instructions within the computer.

Chapter 6:

Four things the interface between the computer and the hardware does:

  1. Facilitates communication between the software running on the computer and the hardware components.
  2. Translates the instructions and commands issued by the software into machine-readable language that the hardware can understand and execute.
  3. Manages and allocates system resources, such as CPU time, memory, and input/output operations, to ensure efficient and optimal performance of the system.
  4. Provides a user interface for users to interact with the computer system and control its behavior through input devices.

Responsibilities of System Software:

  1. Managing and controlling hardware resources.
  2. Providing a platform and environment for application software to run.
  3. Managing system resources such as memory, CPU, and file systems to optimize system performance and ensure stability.
  4. Ensuring security and protection of the system by providing mechanisms for access control, user authentication, and data encryption.

System Software Terms:

  1. System software: Software designed to provide a platform and environment for application software to run, manage and control hardware resources, optimize system performance, and ensure security and stability of the system.
  2. User interface: The part of a software or system that allows users to interact with and control its behavior.
  3. Language services: Software components or tools that enable the development and execution of software programs written in specific programming languages.
  4. Memory managers: Software components that manage the allocation and deallocation of memory resources in a computer system.
  5. Information managers: Software components that manage the storage and retrieval of data and information in a computer system.
  6. I/O managers: Software components that manage the input and output operations of a computer system.
  7. Schedulers: Software components that manage the allocation of system resources, such as CPU time and memory, to processes and tasks running on the system.
  8. Utilities: Software tools or programs that perform specific tasks or functions that are useful for users or system administrators.
  9. Assembly language: A low-level programming language that uses mnemonics and symbols to represent machine instructions and memory addresses.

Functions of an Operating System

  1. Resource Management: The Operating System manages the computer hardware resources and allocates them to the different applications and processes running on the system.
  2. Process Management: The Operating System manages the creation, execution, and termination of processes and threads running on the system, ensuring that each process gets the necessary resources to execute and that the processes do not interfere with each other.
  3. Memory Management: The Operating System manages the computer's memory resources, allocating memory to applications and processes, and freeing memory when it is no longer needed.
  4. Security: The Operating System provides security mechanisms such as user authentication, access control, and data encryption to protect the system and its data from unauthorized access, malicious attacks, and viruses.
  5. User Interface: The Operating System provides a user interface that enables users to interact with the computer and control its behavior.
