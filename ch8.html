Chapter 8: Information Security

Term Description
Information security Practice of safeguarding digital information from unauthorized access, use, disclosure, disruption, modification, or destruction.
Authentication Process of verifying the identity of an entity (such as a person or a device) to establish trust and grant access to resources.
Authorization Process of granting or denying access to resources based on the authenticated identity and the permissions assigned to that identity.
Dual authentication (two-factor authentication) Security mechanism requiring two different forms of authentication to access a resource.
Social engineering Use of psychological manipulation to trick people into divulging sensitive information or performing actions that are not in their best interest.
Virus Type of malware that replicates itself and infects other computer programs, data files, or the boot sector of a hard drive.
Worm Type of malware that spreads itself over a network by exploiting vulnerabilities in operating systems or other software.
Trojan horse/Trojan Type of malware that disguises itself as a legitimate program to trick users into installing and running it, allowing attackers to gain unauthorized access to the affected system.
Ransomware Type of malware that encrypts the victim's files and demands payment (usually in cryptocurrency) in exchange for the decryption key.
Phishing Type of social engineering attack that uses fraudulent emails, websites, or messages to trick users into divulging sensitive information or performing actions that are not in their best interest.
White Hat Ethical hacker who identifies and fixes security vulnerabilities in computer systems and networks.
Black Hat Hacker who uses their skills and knowledge for illegal or unethical purposes, such as stealing sensitive data, disrupting services, or causing damage to computer systems and networks.
Encryption/Decryption Process of transforming plaintext (unencrypted) data into ciphertext (encrypted) data using a mathematical algorithm and a secret key. Decryption is the reverse process of transforming ciphertext back into plaintext using the same algorithm and key.
Cryptography Study and practice of techniques for secure communication in the presence of third parties (such as attackers or eavesdroppers).
Symmetric encryption algorithm Type of cryptographic algorithm that uses the same secret key for both encryption and decryption of data. Examples include Advanced Encryption Standard (AES) and Data Encryption Standard (DES).
Caesar cipher Simple substitution cipher technique that shifts each letter in the plaintext by a fixed number of positions down the alphabet.
Block cipher Type of symmetric encryption algorithm that breaks plaintext into fixed-length blocks and applies a cryptographic transformation to each block using a secret key.
Data Encryption Standard (DES) Widely used symmetric key encryption algorithm developed by IBM in the 1970s that uses a 56-bit key to encrypt plaintext.
Public key system Encryption system that uses a pair of mathematically related keys, one private and one public, for secure communication. The private key is kept secret, while the public key is widely distributed.
Embedded computers Computers that are part of a larger system or device and are designed for a specific function or task. Examples include microcontrollers in appliances, cars, and medical devices.

How a text file can be hidden in a steganographic image:

A text file can be hidden in a steganographic image by manipulating the least significant bit of each pixel to store the text data. This can be done by replacing the least significant bit of the color values in the image with the binary digits of the text file. By doing this, the image will still appear unchanged to the human eye, but the hidden text can be extracted using steganography tools.

Six ways to protect yourself from most cyber attacks:

  1. Keep your operating system and software up to date with the latest security patches and updates.
  2. Use strong and unique passwords for all your online accounts, and enable two-factor authentication whenever possible.
  3. Be cautious when clicking on links or downloading attachments from unknown or suspicious sources, and use anti-virus and anti-malware software to protect your devices.
  4. Use a virtual private network (VPN) when accessing the internet on public Wi-Fi networks or when traveling to protect your online activities from prying eyes.
  5. Back up your important data regularly to protect against data loss from cyber attacks or other disasters.
  6. Educate yourself about common cyber threats and how to protect against them, and practice good cyber hygiene habits such as avoiding oversharing personal information online.
